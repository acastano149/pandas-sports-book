2  El Flujo de Trabajo Profesional: Method Chaining

El ‚ÄúMethod Chaining‚Äù transforma c√≥digo imperativo en pipelines declarativos, como el pipe %>% del Tidyverse.

2.1 1. El Problema del C√≥digo Imperativo

2.1.1 Ejemplo 1.1: El Estilo ‚ÄúNovato‚Äù - ‚ùå NO HACER

En R (anti-patr√≥n):

df_temp <- filter(gps, TipoSesion == 'Partido')
df_temp <- arrange(df_temp, desc(Distancia_Total_m))
result <- select(df_temp, Nombre, Distancia_Total_m)

En Pandas (igual de malo):

¬øPor qu√© es malo?

  • Crea m√∫ltiples variables que ocupan memoria
  • Dif√≠cil saber el estado en cada paso
  • No se lee como una ‚Äúreceta‚Äù secuencial

2.1.2 Ejemplo 1.2: El Estilo Profesional - ‚úÖ RECOMENDADO

En R (Tidyverse):

gps %>%
  filter(TipoSesion == 'Partido') %>%
  arrange(desc(Distancia_Total_m)) %>%
  select(Nombre, Distancia_Total_m) %>%
  head(5)

En Pandas (Method Chaining):

¬øPor qu√© es mejor? Una sola asignaci√≥n, lectura secuencial, par√©ntesis permiten saltos de l√≠nea.

2.1.3 üß† Pr√°ctica 1.1: Tu primer Chain

Convierte este c√≥digo imperativo en un chain profesional:

df_1 = gps[gps['RPE'] > 6]
df_2 = df_1.groupby('Nombre').mean(numeric_only=True)
df_final = df_2[['HSR_m']]
TipVer soluci√≥n 
resultado = (
    gps
    .query("RPE > 6")
    .groupby('Nombre')
    .mean(numeric_only=True)
    [['HSR_m']]
)
print(resultado)

2.2 2. Creaci√≥n de Columnas con .assign()

2.2.1 Ejemplo 2.1: C√°lculos con lambda

En R (Tidyverse):

gps %>%
  filter(Distancia_Total_m > 0) %>%
  mutate(Intensidad = HSR_m / Distancia_Total_m)

En Pandas:

¬øPor qu√© lambda x? x representa el DataFrame en ese punto del chain. Es como . en magrittr.

2.2.2 üß† Pr√°ctica 2.1: Creando columnas

Filtra sesiones donde Duracion_min > 0, crea Metros_por_Minuto, redondea a 2 decimales, ordena de mayor a menor, y muestra los 10 primeros con nombre, fecha y la nueva columna.

TipVer soluci√≥n 
resultado = (
    gps
    .query("Duracion_min > 0")
    .assign(Metros_por_Minuto = lambda x: x['Distancia_Total_m'] / x['Duracion_min'])
    .round(2)
    .sort_values("Metros_por_Minuto", ascending=False)
    .head(10)
    [['Nombre', 'Fecha', 'Metros_por_Minuto']]
)
print(resultado)

¬øQu√© hace cada paso?

  1. .query() ‚Üí Filtra condici√≥n
  2. .assign() ‚Üí Crea nueva columna
  3. .round(2) ‚Üí Redondea todas las num√©ricas a 2 decimales
  4. .sort_values() ‚Üí Ordena descendente
  5. .head(10) ‚Üí Limita a 10 filas
  6. [[...]] ‚Üí Selecciona columnas finales

2.4 üèÜ Mini-Proyecto: Pipeline de An√°lisis de Carga

2.4.1 Pregunta 1: Ratio de Intensidad

Crea un pipeline que filtre Partidos, calcule HSR/Distancia*100 y muestre el top 5.

TipVer soluci√≥n 
(
    gps
    .query("TipoSesion == 'Partido'")
    .assign(Intensidad_Pct = lambda x: x['HSR_m'] / x['Distancia_Total_m'] * 100)
    .nlargest(5, 'Intensidad_Pct')
    [['Nombre', 'Distancia_Total_m', 'HSR_m', 'Intensidad_Pct']]
)

2.4.2 Pregunta 2: Carga de Entrenamiento (RPE √ó Duraci√≥n)

Calcula la Carga (RPE √ó Duracion_min) y encuentra la sesi√≥n con mayor carga.

TipVer soluci√≥n 
(
    gps
    .assign(Carga = lambda x: x['RPE'] * x['Duracion_min'])
    .nlargest(1, 'Carga')
    [['Nombre', 'TipoSesion', 'RPE', 'Duracion_min', 'Carga']]
)

2.4.3 Pregunta 3: Promedio por Jugador (sesiones de alta carga)

Filtra sesiones con RPE >= 7, agrupa por jugador, calcula media de distancia.

TipVer soluci√≥n 
(
    gps
    .query("RPE >= 7")
    .groupby('Nombre')['Distancia_Total_m']
    .mean()
    .sort_values(ascending=False)
)

2.4.4 Pregunta 4: Pipeline Completo con Debugging

Usa .pipe() para mostrar el n√∫mero de filas en cada paso del pipeline anterior.

TipVer soluci√≥n 
def debug(df, paso):
    print(f"Paso '{paso}': {len(df)} filas")
    return df

(
    gps
    .pipe(debug, "Inicial")
    .query("RPE >= 7")
    .pipe(debug, "Filtrado RPE>=7")
    .groupby('Nombre')['Distancia_Total_m']
    .mean()
)
